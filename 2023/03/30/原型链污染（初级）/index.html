原型链污染（初级）

发表于2023-03-30|更新于2023-03-30
|阅读量:

原型链污染，是一种存在于node.js的前端漏洞，因此在了解什么是原型链污染时，我们首先要了解什么是原型链，以及什么是原型。

1、原型对象与原型链

这里，我们首先先说明一下js当中的面向对象，面向对象在我们学习php反序列化的时候曾经学习过，对php来说，面向对象首先要定义一个类，以class为关键词，之后结合new关键词来进行实例化，从而生成一个对象，而在js当中，我们不需要class关键词，而是利用另一种办法。在js中，我们利用函数进行对象的生成和实例化，例如
function Person(name, age)
{

this.name = name;

this.age = age;

this.species = ‘人类’;

this.say = function () {

console.log(“Hello”);

}

}

这是一个简单的构造函数，之所以叫做构造函数，是因为我们利用它来进行一个对象的实例化，因此这种函数通常函数名首字母大写，便于区分。可以看到，我们这里对于对象属性的引用并不像php一样用->，而是采用.的方式。但是这样的构造函数会产生一些问题，也就是在进行实例化的时候，我们会不断地生成一个新函数，这样会不断加剧内存，因此，这里我们就引入了一个新的概念——原型对象。什么是原型对象呢？我们看一个例子

function Person(name, age)
{

this.name = name;

this.age = age;

this.species = ‘人类’;

}
Person.prototype.say = function () {

console.log(“Hello”);
}

这里看到，我们将say函数移到了构造函数的外面，并且引入了一个定义词，也就是显示原型，显示原型就是利用prototype属性查找原型（JS声明构造函数(用来实例化对象的函数)时，会在内存中创建一个对应的对象，这个对象就是原函数的原型。构造函数默认有一个prototype属性，prototype的值指向函数的原型。）。这就是一个原型对象，可以供自己的实例化对象使用，也就是将say这个函数变成了公用的。从其他师傅那边拿到了一张比较好理解的图

原型对象是用来存放实例对象的公有属性和公有方法的一个公共对象。而反过来Person.prototype.construct则表示这个函数，因为.construct也是一个原型。那么他们的基本关系图如下，

那么在知道了这些之后，我们继续，现在我们知道了，Person.prototype实际上也是一个对象型属性，那么他也就拥有这个属性，那么我们现在研究的问题就在于，这个原型对象它的_proto_又指向哪里呢？因此，这里我们需要继续知道一个函数Object()，这个函数是用来实例化所有的对象，那么作为原型对象，他也就自然是由这个函数实例化生成的，因此原型对象的_proto_最终指向的就是Object.prototype,，这就构成了一条原型链，而这个Object()函数的_proto_指向null，也就是空。因此，他们之间的关系图还可以再具体，如下。

在了解了这些之后，我们再来看函数，实际上，所有的函数都是由Function()这个构造函数实例化生成的，那么就产生了一些有意思的东西，Funtion()本身也是一个函数，也就是说Function()是Function()的实例化，Function.proto===Function.prototype,Function.prototype===Object.proto，Function.prototype===Object.prototype。到这里，就是对原型链的基本了解。

2、原型链污染

要想形成原型链污染，首先要满足三个条件

1、 对象递归合并

2、 按路径定义属性

3、 对象克隆

下面，借用一下csdn上面XiLitter师傅的一个例子(感谢师傅，从他的文章学习了很多)，XiLitter师傅举了一个例子
function copy(object1, object2){

for (let key in object2) {

    if (key in object2 && key in object1) {

        copy(object1[key], object2[key])

    }

else {

object1[key] = object2[key]

    }

}

}，题目出自ctfshow的web题，这里满足第一个条件，函数的递归使用，接下来，我们来审计一下代码，首先在object2中寻找键值，如果键值在object1且在object2里面的话，就进行递归，否则，将object2[key]赋值到object1[key]当中，这里，key满足我们的第二个条件，而赋值满足第三个条件，因此我们来看payload，{“proto“:{“ctfshow”:”36dboy”}}

我们首先来进行一个分析，关于_proto_是什么我们之前已经知道了，是一个属性，而在类中我们也可以通过键值的方式来引用属性，比如object[]的形式，这和object.xxx类似。但是我们这里要注意将它构造成json格式的字符串，这样的目的是为了让_proto_成为一个键值，从而进行污染。而我们知道，所有的对象中都存在这个属性，也就是说，在_proto_成为键值的时候，这个if语句一定成立，那么就会进行一个递归，这样的话就会进行第二次的键值寻找，而在第二次中，由于我们的object1的_proto_属性中，不存在这个ctfshow键值，那也就是说我们将会跳出这个if语句，转而将这个属性赋值到object1._proto_当中，而我们知道，object1.proto==Object.prototype，这样就成功进行了一个污染，这样原型链中就存在了我们的一个自定义属性，这样就算我们之后生成一个新的函数，那么这个函数也可以得到我们的自定义属性。这就是关于原型链污染的一个最基本的学习。以上内容的学习全是靠各位师傅的博客学习的，感谢师傅，也附上学习网址https://www.cnblogs.com/20175211lyz/p/12659738.html
