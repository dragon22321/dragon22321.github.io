在学习php反序列化漏洞之前，我们首先先要弄懂一个概念，就是关于php语法中类的概念。

1、php语法中的类

由class来定义

例如：

class a{

var $b=1;

function c(){}

}

这里a表示类的名字,b表示类中的属性名（也可以看作是一个变量，不过在类里面称作属性），1是b这个属性的属性值；而function在类的里面被称作方法。

可以很明显的看出来，类的概念，和我们之前学的c语言中结构体的概念很类似，只不过名称有所改变。因此，和c语言类似，当我们想调用一个类的时候，语法如下

$d= new a;

这就表示了这个类已经被调用了，而如何给其赋值呢？

$a->b=7;

$a->c();

这就是对类的一个调用,也可以叫做实例化，这里的d就是一个对象

当然，上述的调用是最简单的，事实上，类的调用及其复杂，接下来就来学习一个在类的内部经常存在的一些定义和一些变量。

$this

第一次看到类的时候，很自然会被这个符号搞得晕头转向的，但是其实这个$this,其代指的是这个类本身，还是据上面的例子
例如：

class a{

var $b=1;

function c(){}

echo $this->b;

}
$d= new a;

$a->b=7;

$a->c();

这时，会输出一个7，$this此时就相当于被赋值成了d。

类 − 定义了一件事物的抽象特点。类的定义包含了数据的形式以及对数据的操作。

对象 − 是类的实例。

成员变量 − 定义在类内部的变量。该变量的值对外是不可见的，但是可以通过成员函数访问，在类被实例化为对象后，该变量即可成为对象的属性。

成员函数 − 定义在类的内部，可用于访问对象的数据。

在类的定义时，我们有时候可以通过一个初始函数来初始化类，该函数为

void __construct ([ mixed $args [, $… ]] )

而赋值主要用来在创建对象时初始化对象， 即为对象成员变量赋初始值，在创建对象的语句中与 new 运算符一起使用。

例：

function __construct( $par1, $par2 ) {
$this->url = $par1;
$this->title = $par2;
}

通过这个函数，我们在调用这个类的时候就可以获得一个初始化的值，

例如：

$runoob = new Site(‘www.runoob.com', ‘菜鸟教程’)

而在类中，还有一个函数，被称作析构函数

析构函数(destructor) 与构造函数相反，当对象结束其生命周期时（例如对象所在的函数已调用完毕），系统自动执行析构函数。

PHP 5 引入了析构函数的概念，这类似于其它面向对象的语言，其语法格式如下：

void __destruct ( void )

可以理解为， __construct()函数是当开头的时候会调用的函数，而__destruct()函数是结尾时候调用的函数

上面的两个函数是比较常见的两个函数，然而在类中，还存在着对属性的制约，一般来说，有四种定义方式

1 . public属性：在类的外部和内部都可以调用 被继承能够重构（用var定义时也是公有属性）

2 . protected属性：在类的内部可以调用外部不能可以被继承并且重构

3 . private属性：只有在类的内部调用 不能被继承

4 . static属性：类的内部和外部都能调用 内部调用方法(self：：属性变量) 外部调用方法(类名：：属性变量)

所谓的内部调用，就是指在类的内部进行一个值的调用，而外部调用，就是从外部调用类内部的值，而类的继承，运用到的时extended前缀，表示继承的意思，如:

class Child extends Parent {
// 代码部分
},此时，我们可以在子类中进行一个代码的增添，或者是对源代码进行一个修改，这就方便了我们对类的调用。

在学习完php中类的这个概念后，我们就正式地进入了php反序列化的学习。

1、序列化和反序列化的定义

这里我们首先就要知道，什么是php的序列化和反序列化，先来说序列化，在php中我们进行数据传输的时候，市场会利用serialize()函数进行序列化，把数据变成一种可逆的数据结构
这里引用csdn上面一个大佬的例子可以更好的理解：

我们都会在淘宝上买桌子，桌子这种很不规则的东西，该怎么从一个城市运输到另一个城市，这时候一般都会把它拆掉成板子，再装到箱子里面，就可以快递寄出去了，这个过程就类似我们的序列化的过程（把数据转化为可以存储或者传输的形式）。当买家收到货后，就需要自己把这些板子组装成桌子的样子，这个过程就像反序列的过程（转化成当初的数据对象）

而序列化，我们通常是指，将对象，字符串，变量，数组变成一种特定的格式的字符串，使其在数据传输的时候保持稳定。举个例子:
<?php

class Person {

public $name = "Tom";

private $age = 18;

protected $sex = "male";

public function hello() {

    echo "hello";
}

}

$class = new Person();

$class_ser = serialize($class);

echo $class_ser;

?>

输出的结果会是

O:6:”Person”:3:{s:4:”name”;s:3:”Tom”;s:11:”Personage”;i:18;s:6:”*sex”;s:4:”male”;}

其中，o代表对象，6是代表后面的字符串，3是代表其中元素的个数，（注意，在对象中的方法是不会被序列化的）而s代表字符串，i代表整形，中间以；为分隔，代表各个属性所对应的属性名和属性值，而在这里注意若在类中对属性 定义前面加上了protected和private
那么其对应的序列化之后的结果也会不一样，例如:

private，会变成 %00类名%00属性名

$age,变成了%00person%00age，两个%00各占一个个数。而%00不可见

protected，会变成 %00*%00属性名

$sex变成了%00*%00sex

而%00虽然是不可见的，但是当我们提交序列化的结果时还是要提交的

以上内容，就是关于php反序列化时的过程和结果，而接下来的反序列化就是利用unserialize()函数进行反序列化，换言之，如果我们遇到这两个函数，那么极有可能是存在反序列化漏洞的。

2、序列化中的魔法函数

__construct()：构造函数，此函数会在创建一个类的实例时自动调用。

__destruct()：析构函数，此函数会在对象的所有引用都被删除或者类被销毁的时候自动调用。

__sleep()：执行serialize()函数之前，会检查类中是否存在_sleep()方法。如果存在，该方法会先被调用。

__wakeup()：执行unserialize()函数之前，会检查类中是否存在_wakeup()方法。如果存在，则会先调用_wakeup()方法，预先准备对象需要的资源。

（绕过__wakeup这个方法，首先查看php版本，当版本在php5 <5.6.25或者php7 <7.0.10时，序列化字符串中表示对象属性个数的值大于真实属性个数就会跳过__weakup方法）

__toString()：当一个对象被当作一个字符串使用时被调用。例如echo 或者拼接字符串时；此方法必须返回一个字符串，否则会产生 E_RECOVERABLE_ERROR 级别的错误。

__get()：访问不存在的成员变量时调用或者在读取不可访问的属性值的时候，此魔法函数会自动调用。

__set()：设置对象不存在的属性或无法访问(私有)的属性时调用

__call()：在调用不存在的方法时被调用。
